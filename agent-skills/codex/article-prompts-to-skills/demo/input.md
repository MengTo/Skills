## Three small interactions that make an editorial archive feel alive

The reference page uses an oversized catalog of independent stories. Its strongest ideas are not the page layout itself, but three behaviors that can transfer to other products.

### Prompt 1 — Pointer lens

Build a circular lens that follows the pointer over a cover image and reveals an alternate treatment beneath it. Clamp the lens to the media bounds, support touch with a press-and-drag gesture, and expose the alternate image without interaction for reduced-motion users.

### Prompt 2 — Reading rail

Add a fixed rail that shows progress through five chapters. Clicking a marker scrolls to its chapter. The active marker must reflect the chapter nearest the viewport center and remain keyboard accessible.

### Prompt 3 — Archive filters

Create animated topic filters for the story grid. Filtering should preserve document focus, announce the number of visible stories, and avoid rearrangement animation when reduced motion is enabled.

## Source constraints

- Do not copy the article's page layout, title, typography, or color palette.
- Preserve each behavior's input methods and accessibility contract.
- Produce one complete skill package per independent behavior.
