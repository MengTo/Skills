# Game Development Skills

Reusable workflows for playable Three.js and browser-game systems. This family is intentionally separate from `web-design`: use it for simulation, combat, AI, inventory, asset pipelines, performance, and gameplay verification.

- `build-isometric-arpg` — assemble a production-ready isometric action RPG in vertical slices.
- `build-game-camera-controls` — create readable isometric, follow, lock-on, and touch cameras.
- `design-action-combat` — specify, implement, tune, and test readable tactical combat.
- `design-game-encounters` — compose fair action-game arenas, waves, objectives, and rewards.
- `tune-enemy-ai` — build fair, deterministic combat behavior.
- `build-game-inventory` — create atomic inventory, loot, equipment, and persistence flows.
- `create-game-vfx` — build readable, bounded, performance-safe effects.
- `build-game-audio-feedback` — connect player intent and game state to responsive audio cues.
- `author-game-levels` — author playable routes, collision, navigation, and objective spaces.
- `optimize-threejs-games` — profile and improve Three.js game performance.
- `build-mobile-threejs-games` — adapt controls, HUD, quality, and QA for mobile web.
- `test-playable-web-games` — verify complete player journeys with deterministic fixtures and real browser evidence.
- `build-hybrid-game-assets` — choose and integrate imported, procedural, generated, and 2D assets.
- `ship-web-games` — package, deploy, verify, and document exact production releases.
